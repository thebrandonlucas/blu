# Style Rules

- Prefer small, focused changes.
- Keep Elm code idiomatic — use pipelines, pattern matching, and type aliases.
- Use Tailwind utility classes for styling; avoid inline Elm CSS where Tailwind covers it.
- Markdown frontmatter must include `title`, `date`, `tags`, and `layout`.
- Snippets use fake dates (2000-01-XX) for ordering — don't change these without reason.
- Comment why, not what.
